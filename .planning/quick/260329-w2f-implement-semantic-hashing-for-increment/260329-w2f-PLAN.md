    Add `const lexer = @import("lexer.zig");` import to cache.zig.

    Add a new function `pub fn hashSemanticContent(source: []const u8) u64` that:
    1. Creates a Lexer from the source
    2. Iterates all tokens via `lexer.next()` (NOT tokenize -- avoid allocating a list)
    3. For each token, skip `.newline` and `.doc_comment` tokens (semantically irrelevant)
    4. For each remaining token, feed `@intFromEnum(token.kind)` into XxHash3
    5. For tokens that carry semantic value (`.identifier`, `.int_literal`, `.float_literal`, `.string_literal`), also feed `token.text` bytes into the hash
    6. Stop at `.eof`
    7. Return the final hash

    Implementation approach for incremental XxHash3 hashing:
    - XxHash3 only has `hash(seed, input)` (one-shot). For incremental hashing, accumulate a buffer of token kind bytes + text bytes and hash at the end. Alternatively, use a simpler approach: use `std.hash.Wyhash` which supports incremental `update()` calls, or chain XxHash3 by using the previous hash as the seed for the next token.
    - Preferred: Use seed-chaining with XxHash3. Start with seed=0, hash first token's bytes, use result as seed for next token. This avoids allocation and maintains fast hashing.

    Modify `hasChanged` to use `hashSemanticContent` instead of raw byte hashing:
    - Read file content (already done)
    - Call `hashSemanticContent(content)` instead of `XxHash3.hash(0, content)`
    - Compare against cached hash

    Modify `updateHash` similarly:
    - Read file content (already done)
    - Call `hashSemanticContent(content)` instead of `XxHash3.hash(0, content)`
