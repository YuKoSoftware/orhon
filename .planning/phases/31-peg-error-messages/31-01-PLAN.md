<objective>
Extend the PEG engine to accumulate all expected tokens at the furthest failure position and format multi-token expected sets in error messages.

Purpose: Parse errors currently show only the last token alternative tried. Users see "unexpected 'foo'" when the parser could have accepted func, struct, or enum — giving no guidance. This change shows all alternatives.
Output: Modified engine.zig (accumulation + dedup), module.zig (consumer formatting), main.zig (analysis output formatting), plus unit tests in engine.zig.
</objective>

    **1. Add `formatExpectedSet` helper function in `src/module.zig`:**

    Add a file-scope function (not pub, not inside the Resolver struct) near the error formatting code. Import `kindDisplayName` from the engine module. The function takes an allocator and a `[]const TokenKind` slice, returns `![]u8`:

    ```zig
    fn formatExpectedSet(alloc: std.mem.Allocator, set: []const TokenKind) ![]u8 {
        const engine_mod = @import("peg/engine.zig");
        var buf = std.ArrayList(u8).init(alloc);
        errdefer buf.deinit();
        try buf.appendSlice("expected ");
        for (set, 0..) |kind, i| {
            if (i > 0 and i < set.len - 1) try buf.appendSlice(", ");
            if (i > 0 and i == set.len - 1) {
                if (set.len > 2) try buf.appendSlice(", or ") else try buf.appendSlice(" or ");
            }
            try buf.append('\'');
            try buf.appendSlice(engine_mod.kindDisplayName(kind));
            try buf.append('\'');
        }
        return buf.toOwnedSlice();
    }
    ```

    This produces per D-01:
