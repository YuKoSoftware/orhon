The `orhon test` command calls `runTests()` which runs `zig build test` and pipes stderr to `formatTestOutput()`. The issue is in how the output is parsed and displayed.

**Root cause analysis:** In Zig 0.15, `zig build test` output format differs from raw `zig test`. The build system wraps test execution. When all tests pass, `zig build test` may produce minimal or no stderr output. The `formatTestOutput` function (line 273) parses stderr looking for "N/M passed" patterns, but if stderr is empty (all passed), it falls through to the `all_passed` branch which correctly prints "all tests passed".

However, `runTests()` at line 376 passes the argument "test" as a separate arg to `zig build`. Zig 0.15's build system expects `zig build test` where "test" is a build step name. Check this is correct.

The real issue is likely that `formatTestOutput` at line 275 creates a `std.fs.File.stderr().writer(&buf)` — this is a **buffered writer** that writes to the process's stderr, but the caller (`runTests`) checks `result.success` (the exit code from `zig build test`). If the Zig test runner succeeds (exit 0), `all_passed` is true and the "all tests passed" line is printed to stderr. The test script checks the **combined stdout+stderr** via `2>&1`.

Investigate and fix these potential issues:

1. **Check the writer flush**: `formatTestOutput` calls `out.flush()` at line 321, which should work. But verify that `std.fs.File.stderr().writer(&buf)` is the correct Zig 0.15 API — it may have changed. In Zig 0.15, the correct pattern is:
   ```zig
   const stderr_file = std.io.getStdErr();
   var bw = std.io.bufferedWriter(stderr_file.writer());
   const out = bw.writer();
   // ... write ...
   bw.flush() catch {};
   ```
   Update `formatTestOutput` and `printRaw` to use `std.io.getStdErr()` + `std.io.bufferedWriter()` pattern if the current pattern is wrong.

2. **Verify the `zig build test` invocation**: The args array should be `[zig_path, "build", "test"]`. This looks correct for running the test step defined in build.zig. Verify the generated build.zig test step is correct for Zig 0.15 — it should use `b.addTest()` with a root_module that has a `.root_source_file`.

3. **Check stderr vs stdout**: `zig build test` in Zig 0.15 may output test results to **stdout**, not stderr. The `formatTestOutput` function only parses `result.stderr`. If test output goes to stdout, parsing fails and no results are shown. Fix: check both `result.stdout` and `result.stderr` for test output patterns, or use the correct stream.

4. **Add a unit test** for `formatTestOutput`:
   - Test with `all_passed=true` and empty stderr — should not panic
   - Test with sample Zig test stderr containing "N/M passed" and "'module.test.name' failed" patterns

5. **Verify `writer` API**: In Zig 0.15, `std.fs.File.stderr()` may not exist — it may be `std.io.getStdErr()`. Check and fix the writer creation in `formatTestOutput` (line 275) and `printRaw` (line 266).
