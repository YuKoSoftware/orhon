**PEG grammar fix** in `src/orhon.peg`:

At line 364-367, the `unary_expr` rule currently handles only `!` and `&`. Add `-` as a unary prefix operator:

Change:
```
unary_expr
    <- '!' unary_expr
     / '&' unary_expr
     / postfix_expr
```

To:
```
unary_expr
    <- '!' unary_expr
     / '-' unary_expr
     / '&' unary_expr
     / postfix_expr
```

IMPORTANT: Place `-` BEFORE `&` because the PEG `-` must be checked before falling through. The `-` alternative must come after `!` (which is clearly not `-`) and before `&` (address-of). This ensures `-0.5` in an argument list is parsed as `unary_expr(-, float_literal(0.5))` rather than failing because the `add_expr` rule tries to parse it as binary subtraction with no left operand.

**CGN-04: Cross-module `is` operator — two locations to fix**

The bug: `ev is module.Type` where `ev` is a tagged union emits `@TypeOf(ev) == module.Type` which always evaluates to the union's compile-time type, not the runtime active tag.

Fix: Before falling through to the `@TypeOf` path, check if the left-hand value is a tagged union (arbitrary_union TypeClass). If so, emit a tagged union comparison instead.

**Location 1 — AST path** (around line 1770-1778):

The current code for qualified type check `val is module.Type` (when `b.right.* == .field_expr`) unconditionally emits `@TypeOf`. Change it to check if the value is an arbitrary_union first.

Replace the block starting at the comment `// Qualified type check: val is module.Type per D-07` (line 1770):

```zig
// Qualified type check: `val is module.Type`
if (b.right.* == .field_expr) {
    // If left side is a tagged union, emit union tag comparison
    if (self.getTypeClass(val_node) == .arbitrary_union) {
        // Extract the type name (last component of module.Type path)
        const fe = b.right.field_expr;
        try self.emit("(");
        try self.generateExpr(val_node);
        try self.emitFmt(" {s} ._{s})", .{ cmp, fe.name });
        return;
    }
    // Non-union: comptime type check via @TypeOf
    try self.emit("(@TypeOf(");
    try self.generateExpr(val_node);
    try self.emitFmt(") {s} ", .{cmp});
    try self.emitTypePath(b.right);
    try self.emit(")");
    return;
}
```

**Location 2 — MIR path** (around line 2219-2227):

Same fix for the MIR code path. The current code for `rhs_mir.kind == .field_access` unconditionally emits `@TypeOf`.

Replace the block starting at the comment `// Qualified type check: val is module.Type per D-07` (line 2219):

```zig
// Qualified type check: `val is module.Type`
if (rhs_mir.kind == .field_access) {
    // If left side is a tagged union, emit union tag comparison
    if (lhs_mir.type_class == .arbitrary_union or
        val_mir.type_class == .arbitrary_union)
    {
        // Extract the type name from the field access
        const type_name = rhs_mir.name orelse "";
        try self.emit("(");
        try self.generateExprMir(val_mir);
        try self.emitFmt(" {s} ._{s})", .{ cmp, type_name });
        return;
    }
    // Non-union: comptime type check via @TypeOf
    try self.emit("(@TypeOf(");
    try self.generateExprMir(val_mir);
    try self.emitFmt(") {s} ", .{cmp});
    try self.emitTypeMirPath(rhs_mir);
    try self.emit(")");
    return;
}
```

**CGN-05: Async(T) maps to void — one location to fix**

At line ~4159-4160, the `typeToZig` function has:
```zig
} else if (std.mem.eql(u8, g.name, "Async")) {
    break :blk "void"; // Async not yet implemented
```

Replace this with a compile error report. The function already has access to `self.reporter`. Change to:

```zig
} else if (std.mem.eql(u8, g.name, "Async")) {
    const err_msg = try std.fmt.allocPrint(self.allocator, "Async(T) is not yet implemented — cannot use as a type", .{});
    defer self.allocator.free(err_msg);
    try self.reporter.report(.{ .message = err_msg });
    break :blk "void"; // fallback after error
```

This reports a clear compile error instead of silently producing `void`. The `reporter.report()` will cause `hasErrors()` to be true, stopping compilation after the pass completes.
