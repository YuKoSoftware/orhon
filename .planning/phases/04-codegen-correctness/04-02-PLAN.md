fn annotateCallCoercions(self: *MirAnnotator, c: parser.CallExpr) !void {
    const sig = self.resolveCallSig(c) orelse return;
    const param_count = @min(c.args.len, sig.params.len);
    for (c.args[0..param_count], sig.params[0..param_count]) |arg, param| {
        const arg_type = self.lookupType(arg) orelse continue;
        const coercion = detectCoercion(arg_type, param.type_);
        if (coercion.kind) |kind| {
            try self.node_map.put(self.allocator, arg, .{
                .resolved_type = arg_type,
                .type_class = classifyType(arg_type),
                .coercion = kind,
                .coerce_tag = coercion.tag,
            });
        }
    }
}

From src/mir.zig — resolveCallSig (line 557):
```zig
fn resolveCallSig(self: *const MirAnnotator, c: parser.CallExpr) ?declarations.FuncSig {
    // Direct call: func_name(args)
    if (c.callee.* == .identifier) {
        return self.decls.funcs.get(c.callee.identifier);
    }
    // Module call: module.func_name(args) or module.Type.method(args)
    if (c.callee.* == .field_expr) {
        const local = self.decls.funcs.get(c.callee.field_expr.field);
        if (local != null) return local;
        // Cross-module: module.func_name
        if (c.callee.field_expr.object.* == .identifier) {
            if (self.all_decls) |ad| {
                if (ad.get(c.callee.field_expr.object.identifier)) |mod_decls| {
                    return mod_decls.funcs.get(c.callee.field_expr.field);
                }
            }
        }
        // Cross-module: module.Type.method
        if (c.callee.field_expr.object.* == .field_expr) {
            const inner = c.callee.field_expr.object.field_expr;
            if (inner.object.* == .identifier) {
                if (self.all_decls) |ad| {
                    if (ad.get(inner.object.identifier)) |mod_decls| {
                        return mod_decls.funcs.get(c.callee.field_expr.field);
                    }
                }
            }
        }
    }
    return null;
}
```

**CGEN-02 fix in src/mir.zig:**

The `annotateCallCoercions` function at line 437 already works correctly for ALL calls — it calls `resolveCallSig(c)` which already handles cross-module lookup via `all_decls`. The coercion detection (`detectCoercion`) then checks if arg type differs from param type (including `value_to_const_ref` when param is `const &T` and arg is `T`).

The issue is that `resolveCallSig` returns `null` for some cross-module call patterns. Specifically, struct instance method calls like `obj.method(args)` where `obj` is a variable of a cross-module struct type. In this pattern:
- `c.callee` is a `field_expr` where `.field` is the method name and `.object` is the variable `obj`
- `obj` is an `.identifier` but its name is the variable name, not the module name
- So the cross-module branch `ad.get(c.callee.field_expr.object.identifier)` looks up the variable name in `all_decls`, which fails

The fix: In `resolveCallSig`, after the cross-module lookups fail, add a fallback that looks up the method name in ALL modules' DeclTables when the callee is a field_expr and the object is a non-module identifier (instance method call pattern). Add this BEFORE the final `return null` at line 587:

```zig
// Instance method call: obj.method(args) where obj's type is from another module.
// The object is a variable, not a module name. Try all modules for the method.
if (c.callee.* == .field_expr) {
    const method_name = c.callee.field_expr.field;
    // Already checked local decls above; try cross-module
    if (self.all_decls) |ad| {
        var it = ad.iterator();
        while (it.next()) |entry| {
            if (entry.value_ptr.*.funcs.get(method_name)) |fsig| {
                return fsig;
            }
        }
    }
}
```

Wait — this is too broad and could match wrong overloads. A better approach: the MIR annotator has `type_map` which maps nodes to resolved types. Look up the object's type, extract the module name from the struct type, then look up the method in that module's DeclTable.

Actually, re-examine the issue. `resolveCallSig` takes a `parser.CallExpr` (AST-level). At the MIR annotation stage, the type_map IS populated. But `resolveCallSig` doesn't use type_map. The simplest correct fix:

In `resolveCallSig`, when the callee is `field_expr` and the object is an identifier that's NOT a module name, look up the object's type in `type_map`. If the type is `.struct_type` with a module-qualified name (e.g., "tester.Counter"), extract the module name and look up the method there.

Add after the existing cross-module checks (after line 585), before `return null`:

```zig
// Instance method: obj.method(args) — resolve obj's type to find the module
if (c.callee.* == .field_expr) {
    const obj_type = self.lookupType(c.callee.field_expr.object) orelse return null;
    // struct_type carries the struct name; check if any module has this method
    if (self.all_decls) |ad| {
        var it = ad.iterator();

            if (entry.value_ptr.*.funcs.get(c.callee.field_expr.field)) |fsig| {

The key principle: when `resolveCallSig` returns non-null, `annotateCallCoercions` correctly applies `value_to_const_ref`. The fix is making `resolveCallSig` find the signature for cross-module instance method calls. Per D-04 and D-05.
