  <action>
    Per D-03, D-04, D-05, fix both interpolation codegen functions to emit defer free after allocPrint.

    **Context:** Both `generateInterpolatedString` (AST path, line 2580) and `generateInterpolatedStringMir` (MIR path, line 2983) emit `std.fmt.allocPrint(...)` as an inline expression with no corresponding free. The MIR hoisted path (temp_var + injected_defer at lines 1433-1447) already handles this correctly. Both non-hoisted paths are currently dormant (PEG builder does not yet generate interpolation nodes), but must be fixed proactively per D-05.

    **Challenge:** Both functions are called from expression-generation contexts (inside `generateExpr` and `generateExprMir` switch arms). You cannot emit statement-level `const` + `defer` lines from within an expression fragment.

    **Solution — pre-statement hoisting buffer:**

    1. Add two fields to the CodeGen struct:
       - `interp_count: u32` — initialized to 0, incremented for each temp var
       - `pre_stmts: std.ArrayList(u8)` — buffer for hoisted declarations, initialized in `init()`, deinited in `deinit()`

    2. Modify `generateInterpolatedString` (AST path):
       - Increment `interp_count` to get a unique N
       - Build the temp var name `_interp_N` (use `std.fmt.bufPrint` into a stack buffer)
       - Append to `pre_stmts`: the current indentation + `const _interp_N = std.fmt.allocPrint(std.heap.page_allocator, "fmt", .{args}) catch |err| return err;\n`
       - Append to `pre_stmts`: the current indentation + `defer std.heap.page_allocator.free(_interp_N);\n`
       - Emit just `_interp_N` to the main output (replacing the inline allocPrint)

    3. Modify `generateInterpolatedStringMir` (MIR non-hoisted path) — same pattern as step 2.

    4. Add a `flushPreStmts` helper method that writes `pre_stmts` contents to the main output buffer and clears it.

    5. Call `flushPreStmts()` at the start of each statement emission:
       - In `generateStatement` (AST path) — at the top, before the switch
       - In `generateStatementMir` (MIR path) — at the top, before the switch
       This ensures hoisted declarations appear on their own lines before the statement that references them.

    6. Add a unit test in codegen.zig that verifies the `pre_stmts` mechanism works — or, since the interpolation paths are dormant, add a source-level regression test: `grep -c "defer std.heap.page_allocator.free" src/codegen.zig` should find the emit strings in the code.

    **Important:** The existing MIR hoisted path (temp_var + injected_defer) must continue to work unchanged. Only the non-hoisted call sites change behavior.
  </action>
