# Requirements: Orhon Compiler

**Defined:** 2026-03-28
**Core Value:** A clean, correct compiler with zero workarounds — every bug fixed, every error propagated, every code path honest.

## v0.16 Requirements

Requirements for v0.16 Bug Fixes milestone. Each maps to roadmap phases.

### Codegen

- [x] **CGN-01**: `const &BridgeStruct` parameter emits pointer pass (`&arg`) instead of by-value
- [x] **CGN-02**: Bridge struct value params in error-union functions stay by-value (no silent `*const` promotion)
- [x] **CGN-03**: Sidecar `export fn` generates `pub export fn` so bridge functions are accessible
- [x] **CGN-04**: Cross-module `is` operator emits tagged union check instead of `@TypeOf` comparison
- [x] **CGN-05**: `Async(T)` reports compile error instead of silently mapping to `void`

### Parser

- [x] **PRS-01**: Negative float/int literals accepted as function call arguments (`-0.5`, `-1`)

### Build System

- [x] **BLD-01**: Multi-file module with Zig sidecar resolves without "file exists in two modules" error
- [x] **BLD-02**: `#cimport` bridge file adds include path for module-relative headers
- [x] **BLD-03**: `#cimport source:` generates `linkSystemLibrary` for owning module
- [x] **BLD-04**: Cross-compilation `-win_x64` passes valid step name to Zig build
- [x] **BLD-05**: `orhon build -fast` uses standard cache directories (no leak into `bin/`)

### Documentation

- [x] **DOC-01**: TODO.md updated — mark 4 fixed bugs as fixed (cast_to_enum, null_multi_union, empty_struct, size keyword)
- [x] **CLN-01**: Remove `Async(T)` from grammar and codegen — unimplemented, dead weight

## Future Requirements

None deferred — all known bugs included in v0.16.
