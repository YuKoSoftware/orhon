# Phase quick-260330-04a Plan 01: Comma-Separated Libraries in #cimport Summary

**One-liner:** Comma-split #cimport name field at all 3 pipeline extraction sites via shared splitCimportLibNames helper, with per-segment duplicate detection and whitespace trimming.

## What Was Built

Support for comma-separated library names in the `#cimport` `name:` field. Users can now write:

```orhon
#cimport = { name: "vulkan, SDL3", include: "vulkan/vulkan.h" }
```

instead of needing separate `#cimport` blocks for each library. The compiler splits the name on commas, trims whitespace from each segment, skips empty segments, and emits a `linkSystemLibrary` call for each individual library in the generated `build.zig`.

## Implementation Details

### Helper function

`splitCimportLibNames(allocator, name, out)` in `src/pipeline.zig`:
- Uses `std.mem.splitScalar(u8, name, ',')` to iterate comma-separated segments
- Trims each segment with `std.mem.trim(u8, segment, " \t")`
- Skips empty segments after trimming
- Appends non-empty trimmed slices (slices into the original string — no extra allocation)

### 3 extraction sites updated

1. **Multi-target `collectCimport` helper (~line 608):** Duplicate detection now runs per split segment, registering each individual library name.
2. **Single-target root module (~line 779):** Replaced direct append with split loop.
3. **Single-target non-root modules (~line 817):** Replaced linear-scan duplicate check on `lib_name` with per-segment check.

### Unit tests (5)

- Single name: `"SDL3"` → `["SDL3"]`
- Two names with space: `"vulkan, SDL3"` → `["vulkan", "SDL3"]`
- Two names no spaces: `"vulkan,SDL3"` → `["vulkan", "SDL3"]`
- Extra whitespace: `" vulkan , SDL3 "` → `["vulkan", "SDL3"]`
- Empty segments skipped: `"a,,b"` → `["a", "b"]`
