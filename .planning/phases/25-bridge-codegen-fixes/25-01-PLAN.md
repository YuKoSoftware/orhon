Per CGN-03: When copying sidecar .zig files to the generated directory, `export fn` declarations lack `pub` visibility. The generated main.zig does `@import("mod_bridge").func_name` which requires `pub` on the symbol.

Currently (main.zig ~line 1110): `std.fs.cwd().copyFile(sidecar_src, std.fs.cwd(), sidecar_dst, .{})` — a verbatim copy.

Replace the `copyFile` call with a read-modify-write that ensures all `export fn` have `pub` visibility:

1. Read the sidecar file content: `const content = try std.fs.cwd().readFileAlloc(allocator, sidecar_src, 1024 * 1024);`
2. Scan for `export fn` that is NOT preceded by `pub `. For each occurrence of `export fn`, check if it's at position 0 or preceded by a newline (not preceded by `pub `). Replace with `pub export fn`.
3. Write the modified content to `sidecar_dst`.

Implementation approach — use a simple scan-and-replace loop:
```zig
// Read sidecar content
const content = try std.fs.cwd().readFileAlloc(allocator, sidecar_src, 1024 * 1024);
defer allocator.free(content);

// Ensure all `export fn` have pub visibility
var result = std.ArrayListUnmanaged(u8){};
defer result.deinit(allocator);
var pos: usize = 0;
while (std.mem.indexOfPos(u8, content, pos, "export fn")) |idx| {
    // Check if already preceded by "pub "
    const already_pub = idx >= 4 and std.mem.eql(u8, content[idx - 4 .. idx], "pub ");
    try result.appendSlice(allocator, content[pos..idx]);
    if (!already_pub) {
        try result.appendSlice(allocator, "pub ");
    }
    pos = idx;
}
try result.appendSlice(allocator, content[pos..]);

// Write modified sidecar
const dst_file = try std.fs.cwd().createFile(sidecar_dst, .{});
defer dst_file.close();
try dst_file.writeAll(result.items);
```

This replaces the single `copyFile` line at main.zig ~1110 with the read-modify-write block above. Keep the surrounding `if (mod_ptr.has_bridges)` and `if (mod_ptr.sidecar_path)` guards unchanged.
