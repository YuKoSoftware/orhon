  <name>Task 1: Fix intermittent unit test failure in module.zig</name>
  <files>src/module.zig</files>
  <read_first>src/module.zig (lines 996-1016 — the "read module name" test)</read_first>
  <action>
The root cause of the intermittent failure (RELY-01) is the test "read module name" at
line 996 of module.zig. It writes to a hardcoded `/tmp/test_module.orh` path, which
races when Zig runs tests in parallel threads or when multiple `zig build test`
invocations overlap.

Fix: Replace the hardcoded `/tmp/test_module.orh` with `std.testing.tmpDir` which
provides an isolated temporary directory per test:

1. Replace lines 996-1016 with a test that uses `std.testing.tmpDir(.{})` to get
   a unique temp directory handle.
2. Use `tmp.dir.createFile("test_module.orh", .{})` to create the file inside
   the isolated directory.
3. Build the full path for `readModuleName` using `tmp.dir.realpathAlloc()` or
   by passing the dir-relative path.
4. Call `defer tmp.cleanup()` to auto-clean the temp directory.

The key change:
- OLD: `const tmp_path = "/tmp/test_module.orh";` with manual createFile/deleteFile
- NEW: `var tmp = std.testing.tmpDir(.{});` with `defer tmp.cleanup();`

Since `readModuleName` takes an absolute path string, use `tmp.dir.realpathAlloc`
to get the absolute path of the created file, or construct it from the tmpDir path.

Important: `readModuleName` opens the file via `std.fs.cwd().openFile(path, .{})`,
so the path must be absolute or cwd-relative. Use `realpathAlloc` on the tmpDir
to construct the full path.

Do NOT change the `readModuleName` function itself — only the test.
  </action>

  </verify>

    - grep "/tmp/test_module.orh" src/module.zig returns NO matches
    - `zig build test` passes (exit code 0)
  </acceptance_criteria>
  <done>
    The "read module name" test uses std.testing.tmpDir instead of a hardcoded /tmp path.
    No more race condition on parallel test execution.
